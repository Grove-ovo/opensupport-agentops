# Phase 5F Reports And Integration

## Scenario: Reproducible Benchmark, Load, And Cost Evidence

### 1. Scope / Trigger

- Trigger: changes to Phase 5 report fixtures, report rendering, budget
  interpretation, child-task resolution, or parent integration validation.
- Applies to `scripts/phase5-report-fixtures.mjs`, Phase 5 report generators,
  validators, and the three committed Phase 5 reports.

### 2. Signatures

```text
npm run reports:phase5
npm run reports:phase5:check
npm run test:phase5
node scripts/validate-phase5.mjs --final
```

### 3. Contracts

- Benchmark and cost reports use the same public V0-V3 runner/comparison
  fixture.
- Load reports execute the existing V3 pipeline through
  `ApplicationLoadHarness` for concurrency 1, 5, 10, and 25.
- Report fixtures inject deterministic timing and event-loop values and state
  that results are not wall-clock or production capacity claims.
- Cost reports keep estimated average/total cost separate from configured
  per-ticket/daily tenant budgets and calculate both headroom values.
- V3 relative cost is `(V3 average - variant average) / variant average`.
- Phase 5 integration resolves child tasks in either active or archive
  locations and verifies exact parent linkage and executable commands.

### 4. Validation & Error Matrix

| Condition | Behavior |
|---|---|
| Report differs from generated bytes | report check fails |
| Missing metric, budget, or limitation label | Phase 5F validation fails |
| Missing/misordered/unlinked child | Phase 5 integration fails |
| Child incomplete in final mode | `validate-phase5 --final` fails |
| Child lacks its own test command | Phase 5 integration fails |

### 5. Good / Base / Bad Cases

- Good: all reports reproduce and all six children resolve from archives.
- Base: 5F is active during its own check; normal integration allows that one
  task while final mode requires completion.
- Bad: deriving budget from measured cost; budget is configuration and must
  remain a separate field.

### 6. Tests Required

- Generate all reports and compare them byte-for-byte.
- Assert load report has all four scenarios, counts, throughput, percentiles,
  and event-loop metrics.
- Assert cost report has measured average/total, both configured budgets,
  headroom, and V3 relative deltas.
- Run integration before and after 5F archive; use `--final` after archive.
- Run full tests, lint, typecheck, and Trellis task validation.

### 7. Wrong vs Correct

#### Wrong

```ts
const dailyBudget = measuredTotalCost;
```

#### Correct

```ts
const dailyHeadroom = configuredDailyBudget - measuredTotalCost;
```

Measured estimates and configured governance limits remain independently
auditable.
