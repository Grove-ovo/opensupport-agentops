## Scope

OpenSupport AgentOps. The run exercised the public Nginx edge, Fastify API,
PostgreSQL, Redis, worker health, deterministic OIDC/provider/Chatwoot mock,
signed Chatwoot ingress, authenticated operator reads, and observability

a public internet capacity claim, live provider benchmark, live Chatwoot SaaS

### Local Sandbox Notes

- The managed sandbox could not run `sudo chown` in
  `scripts/ops/prepare-compose-secrets.sh`. For this local-only run, ignored
  ephemeral CI secret files were temporarily made container-readable and then
  restored to `0600` after teardown. The committed production flow still
  requires the documented ownership preparation step.
- Docker buildx could not write to `~/.docker`; the run used a temporary
  `DOCKER_CONFIG` under `tmp/docker-config`.
- A stale local Docker volume initially caused PostgreSQL password mismatch
  with the newly generated `.env.ci.smoke`. The run was repeated after
  `docker compose down -v --remove-orphans`, matching ephemeral CI semantics.

## Findings

- The new production HTTP load gate successfully validates the real local edge
  path and emits secret-safe JSON/Markdown evidence.
- The production-style stack is healthy under the CI-safe profile and under
  paced sustained profiles below configured Nginx rate limits.
- Burst profiles intentionally exceed configured edge limits and return
  `429 rate_limited` rather than timing out or degrading internally.
- No timeout was observed in any profile. The blocked burst profiles were
  threshold failures caused by edge rate limiting, not application latency.
- Observability remained provisioned after load: Prometheus scraped API/worker
  jobs and Grafana reported the configured datasource/dashboard.
